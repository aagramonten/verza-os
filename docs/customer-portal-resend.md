# Customer portal magic links with Resend

## Current status

Email delivery is **not enabled by default**. The application uses
`NoopMagicLinkSender` unless `CUSTOMER_MAGIC_LINK_PROVIDER=resend` is set with
all required production configuration.

The Resend adapter is ready, but activating provider credentials and domain
DNS is an operational step. Do not enable it in tests or local development.

## Required environment variables

Set these values in the API service environment:

```env
CUSTOMER_MAGIC_LINK_PROVIDER=resend
CUSTOMER_PORTAL_URL=https://cotizar.verzagarden.com
CUSTOMER_EMAIL_FROM=Verza Garden <acceso@correo.verzagarden.com>
RESEND_API_KEY=re_...
```

Related controls remain configurable:

```env
CUSTOMER_MAGIC_LINK_TTL_MIN=15
CUSTOMER_SESSION_TTL_DAYS=30
CUSTOMER_AUTH_RATE_LIMIT_PER_MIN=5
TRUST_PROXY_HOPS=1
```

Rules enforced at startup:

- `RESEND_API_KEY` and `CUSTOMER_EMAIL_FROM` are required when the provider is
  `resend`.
- `CUSTOMER_PORTAL_URL` must use HTTPS in production and cannot contain URL
  credentials.
- The provider defaults to `noop`, so missing production activation cannot
  accidentally send email.

## Domain and Resend setup

1. Add a dedicated sending subdomain in Resend, for example
   `correo.verzagarden.com`.
2. Add the SPF and DKIM records generated by Resend to the DNS provider.
3. Add DMARC when the domain is ready for it.
4. Wait until Resend reports the domain as verified.
5. Create an API key with sending-only access, restricted to the verified
   domain.
6. Set `CUSTOMER_EMAIL_FROM` to an address on that verified domain.
7. Keep click tracking disabled for authentication emails. Rewriting an
   authentication URL adds unnecessary handling of a sensitive one-time link.
8. Configure the variables in Railway and restart the API.
9. Perform one controlled smoke test with a real customer test record. Confirm
   that the request response stays generic and that the link works only once.

## Adapter injection

The composition root is
`apps/api/src/modules/customer-auth/index.ts`.

`createCustomerAuthModule` resolves the sender in this order:

1. `CustomerAuthModuleOverrides.sender`, used by tests and controlled
   composition.
2. `ResendMagicLinkSender` when
   `CUSTOMER_MAGIC_LINK_PROVIDER=resend`.
3. `NoopMagicLinkSender` for every other environment.

`CustomerAuthService` owns token creation and calls the `MagicLinkSender` port.
The infrastructure adapter is
`apps/api/src/modules/customer-auth/infrastructure/resend-magic-link.sender.ts`.
It calls the fixed Resend HTTPS endpoint with native Node `fetch`; the provider
base URL is intentionally not configurable.

## Raw-token handling

The raw magic token is secret and single-purpose:

- The database stores only its SHA-256 hash.
- The public request endpoint always returns the same generic response and
  never returns the token.
- Audit data contains identifier/IP hashes and delivery acceptance metadata,
  never the token.
- Application logs, errors and telemetry must never include the outbound
  request body, provider authorization header, email access URL or raw token.
- Provider errors are translated to sanitized internal errors; provider
  response bodies are not logged or returned.
- The Resend idempotency key contains a SHA-256 token hash, never the raw token.
- The email places the token in the URL fragment:
  `/mi-jardin/verificar#token=...`.
- Angular removes the fragment from browser history before verification and
  never persists the magic token. Only the resulting revocable customer
  session is stored.

Resend accepting the API request is not proof of final inbox delivery. Delivery
webhooks and notification delivery tracking are intentionally deferred.

## Official quote delivery (next activation)

The admin action is intentionally labeled **Marcar como enviada** until the
quote delivery worker is enabled. The production implementation must generate
the PDF from the approved quote snapshot, store it in private tenant-scoped
storage, and send a portal link through Resend. The raw magic token and any
quote access token must never appear in logs, responses, audit payloads, or
telemetry.

Additional production variables will be required:

```env
QUOTE_EMAIL_PROVIDER=resend
QUOTE_EMAIL_FROM=Verza Garden <cotizaciones@correo.verzagarden.com>
QUOTE_EMAIL_REPLY_TO=...
QUOTE_DOCUMENT_STORAGE=r2
QUOTE_DOCUMENT_BUCKET=...
QUOTE_DOCUMENT_PUBLIC_URL= # leave empty; documents are private
```

The sender domain must be verified in Resend with SPF/DKIM records before the
provider is enabled. `SENT` must mean that Resend accepted the request; inbox
delivery requires a later signed webhook integration. Local development keeps
the sender disabled and uses private local storage only for tests.
